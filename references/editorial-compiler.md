# Editorial Compiler

编辑编译器负责把合格叶节点转化成平台内容。它不负责产生思想。

## 一、先选择目标行为

### 点击 / 进入

适合：具体冲突、身份识别、反常现象、明确收益。

### 看完 / 停留

适合：悬念、逐层揭示、案例推进、清晰结构。

### 收藏

适合：模板、步骤、检查表、资源、可复用框架。

### 分享

适合：强身份表达、替受众说出难以表达的话、社会观察。

### 评论

适合：真实分歧、边界问题、角色冲突、需要经验补充的问题。

### 关注

适合：明确系列承诺、持续稀缺价值、可预期的后续内容。

### 私信 / 线索 / 成交

适合：高匹配问题、清晰资格筛选、自然下一步。

## 二、包装结构

- 反常识：打破默认假设，但必须解释机制；
- 痛点解决：具体场景 + 可执行方法；
- 避坑：失败机制 + 识别信号 + 替代动作；
- 教程：明确结果 + 步骤 + 边界；
- 清单：压缩认知，但不能只是堆项目；
- 案例拆解：结果 + 关键变量 + 可迁移机制；
- 争议讨论：真实冲突，不制造假对立；
- 身份叙事：独特处境，不是空贴标签；
- 情绪共鸣：说出矛盾，而不是泛泛煽情；
- 信息差：提供受众看不到的流程或判断标准；
- 工具模板：可直接使用；
- 系列追更：明确这条是内容宇宙中的哪个分支。

## 三、标题检查

一个标题至少做到两件事：

- 让目标受众识别“这是给我的”；
- 准确承诺正文会兑现的价值。

不要为了点击损害内容支柱的长期信任。

## 四、系列化检查

同一系列的下一条必须新增：

- 新机制；或
- 新角色；或
- 新边界；或
- 新场景；或
- 新方法；或
- 新反例。

仅替换名词不算新增内容。
